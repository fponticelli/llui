# LLui Agent — Plan 10: End-to-End Test Harness

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development.

**Goal:** Prove v1 works from Claude's perspective: a real headless browser mounts a minimal LLui app with agent integration, the bridge runs in-process with an MCP client pair, the server runs in-process on an ephemeral http port, and tests exercise every primitive (mint, LAP endpoints, confirm flow, wait_for_change, query_dom, revoke) end-to-end.

**Architecture:** Single new package `packages/agent-e2e/` (kept out of the main agent packages to contain Playwright + its heavy browser download). Test topology per scenario:

```
┌───────────────────────┐
│  test (vitest)        │
│  ┌───────────────┐    │
│  │ MCP Client    │    │   @modelcontextprotocol/sdk Client + InMemoryTransport
│  │    ↕          │    │
│  │ llui-agent    │    │   createBridgeServer from packages/agent-bridge
│  │  bridge       │    │
│  │    ↕ fetch    │    │
│  │ agent server  │    │   createLluiAgentServer on ephemeral Node http port
│  │    ↕ ws       │    │
│  │ browser (pw)  │    │   Playwright launches headless Chromium;
│  │               │    │   loads an inline HTML that imports test-host.ts
│  └───────────────┘    │   which creates the AppHandle + AgentClient
└───────────────────────┘
```

Browser-side: a tiny page that imports a pre-bundled test host via `esbuild` at test startup. Host mounts a `Counter`-like app with annotated Msgs and wires `createAgentClient`. Test assertions reach into state via MCP calls and into DOM via Playwright.

**Tech Stack:** `@playwright/test`, `@modelcontextprotocol/sdk` (InMemoryTransport), `esbuild` for the browser-side bundle, vitest for the orchestrator (NOT playwright-test — we drive Playwright imperatively so the test is a normal vitest file).

**Spec section coverage:** the whole agent spec, but exercised as a living test rather than documented.

**Non-goals:** CI integration (local-only for v1 validation). Multiple-app-per-chat scenarios. Cross-origin scenarios (same origin sufficient). Performance benchmarks.

## Task 2: Browser-side host fixture

**Files:**
- Create: `packages/agent-e2e/src/host.ts`

A minimal LLui app that exercises the agent primitives:

```ts
import { component, mountApp, div, button, text, type Msg as _M } from '@llui/dom'
import {
  createAgentClient, agentConnect, agentConfirm, agentLog,
} from '@llui/agent/client'

type State = {
  count: number
  lastDelete: string | null
  agent: {
    connect: agentConnect.AgentConnectState
    confirm: agentConfirm.AgentConfirmState
    log: agentLog.AgentLogState
  }
}

type Msg =
  /** @intent("Increment the counter") */
  | { type: 'inc' }
  /** @intent("Decrement the counter") */
  | { type: 'dec' }
  /** @intent("Reset to zero") */
  | { type: 'reset' }
  /** @intent("Delete an item") @requiresConfirm */
  | { type: 'delete'; id: string }
  /** @intent("Sign out") @humanOnly */
  | { type: 'signOut' }
  /** @intent("Navigate") @alwaysAffordable */
  | { type: 'nav'; to: 'home' | 'reports' }
  | { type: 'agent'; sub: 'connect'; msg: agentConnect.AgentConnectMsg }
  | { type: 'agent'; sub: 'confirm'; msg: agentConfirm.AgentConfirmMsg }
  | { type: 'agent'; sub: 'log'; msg: agentLog.AgentLogMsg }

const App = component<State, Msg, never>({
  name: 'TestApp',
  init: () => [{
    count: 0,
    lastDelete: null,
    agent: {
      connect: agentConnect.init({ mintUrl: '/agent/mint' })[0],
      confirm: agentConfirm.init()[0],
      log: agentLog.init()[0],
    },
  }, []],
  update: (s, m) => {
    switch (m.type) {
      case 'inc': return [{ ...s, count: s.count + 1 }, []]
      case 'dec': return [{ ...s, count: s.count - 1 }, []]
      case 'reset': return [{ ...s, count: 0 }, []]
      case 'delete': return [{ ...s, lastDelete: m.id }, []]
      case 'signOut': return [s, []]  // humanOnly: agent can't reach this
      case 'nav': return [s, []]
      case 'agent': {
        switch (m.sub) {
          case 'connect': {
            const [next] = agentConnect.update(s.agent.connect, m.msg, { mintUrl: '/agent/mint' })
            return [{ ...s, agent: { ...s.agent, connect: next } }, []]
          }
          case 'confirm': {
            const [next] = agentConfirm.update(s.agent.confirm, m.msg)
            return [{ ...s, agent: { ...s.agent, confirm: next } }, []]
          }
          case 'log': {
            const [next] = agentLog.update(s.agent.log, m.msg)
            return [{ ...s, agent: { ...s.agent, log: next } }, []]
          }
        }
      }
    }
  },
  view: ({ send, text: t }) => [
    div({ 'data-agent': 'root' }, [
      div({ 'data-agent': 'count-display' }, [t((s) => `count: ${String(s.count)}`)]),
      button({ 'data-agent': 'inc', onClick: () => send({ type: 'inc' }) }, [t('+')]),
      button({ 'data-agent': 'dec', onClick: () => send({ type: 'dec' }) }, [t('-')]),
      button({ 'data-agent': 'reset', onClick: () => send({ type: 'reset' }) }, [t('reset')]),
      button({ 'data-agent': 'delete-42', onClick: () => send({ type: 'delete', id: '42' }) }, [t('delete 42')]),
    ]),
  ],
  agentAffordances: (s) => [
    { type: 'nav', to: 'home' },
    { type: 'nav', to: 'reports' },
  ],
  agentDocs: {
    purpose: 'Counter with deletion — used by the LLui Agent e2e test harness.',
    overview: 'Msg variants: inc / dec / reset / delete (requiresConfirm) / signOut (humanOnly) / nav (alwaysAffordable).',
  },
  agentContext: (s) => ({
    summary: `Count is ${s.count}; lastDelete is ${s.lastDelete ?? 'null'}.`,
    hints: [],
  }),
})

// Boot
const root = document.getElementById('app')!
const handle = mountApp(root, App)

const client = createAgentClient({
  handle,
  def: App,
  rootElement: root,
  slices: {
    getConnect: (s: State) => s.agent.connect,
    getConfirm: (s: State) => s.agent.confirm,
    wrapConnectMsg: (m) => ({ type: 'agent', sub: 'connect', msg: m } as Msg),
    wrapConfirmMsg: (m) => ({ type: 'agent', sub: 'confirm', msg: m } as Msg),
  },
})

// Chain the effect handler into a handleEffects chain (or direct — since we have no other effects):
// For the e2e test we'll manually route effects via client.effectHandler.
// The runtime requires an onEffect on the component; we'll add a wrapper later if needed.
// For now, since the component has E=never, no effects fire except agent's.
// BUT the agentConnect.update DOES return effects. Those need handleEffects wired.
// For e2e simplicity, the update() above passes [] for agent-sub updates, which is WRONG.
// Fix: re-emit the sub-component's effects via the component's onEffect.

// For v1 e2e, take shortcut: globally expose client so the test harness can call mint() directly.
(globalThis as any).__lluiE2eClient = client
(globalThis as any).__lluiE2eHandle = handle
client.start()
```

**Note on the effect wiring shortcut:** the component's `onEffect` is missing, meaning AgentEffect values returned by `agentConnect.update` are dropped. For v1 e2e, tests bypass the in-app "Connect with Claude" click and call `client.effectHandler` directly via the exposed `__lluiE2eClient` global. This shortcut acknowledges the rough edge; a follow-up cleans it up by adding the proper `handleEffects` chain to the host.

### build.ts

```ts
import { build } from 'esbuild'
import { fileURLToPath } from 'node:url'
import { dirname, resolve } from 'node:path'

const here = dirname(fileURLToPath(import.meta.url))

/**
 * Bundle host.ts through @llui/vite-plugin's transform so the
 * compile-time metadata (__msgAnnotations, __bindingDescriptors,
 * __schemaHash) is emitted in prod mode (agent: true).
 *
 * For v1 e2e, we use esbuild for speed and manually invoke
 * transformLlui on the source before bundling. This keeps the test
 * runner lightweight (no Vite dev server).
 */
export async function bundleHost(): Promise<string> {
  const { transformLlui } = await import('@llui/vite-plugin/dist/transform.js')
  // Read host.ts, transform, then hand to esbuild as a virtual source.
  // ... (implementation detail: use esbuild's stdin plugin to feed transformed source)
  // Simpler: write a pre-transformed version to a .tmp file and bundle it.
  // This exact implementation is left to the subagent's judgment; fall back to
  // raw esbuild bundling without the vite-plugin transform if it proves too fussy.
  // In that case, the e2e host won't have agent metadata emitted — tests must
  // pre-seed __msgAnnotations etc. on the component def at runtime.

  const result = await build({
    entryPoints: [resolve(here, 'host.ts')],
    bundle: true,
    format: 'iife',
    platform: 'browser',
    target: 'es2022',
    write: false,
  })
  return result.outputFiles[0].text
}
```

**Pragmatic fallback:** if integrating `transformLlui` into esbuild proves fussy, manually build metadata and attach to `App` at runtime in host.ts. The subagent should decide based on what's practical within the task time budget. Document the choice in the commit message.

### harness.ts

```ts
import { chromium, type Browser, type Page } from '@playwright/test'
import { createLluiAgentServer, InMemoryTokenStore, type AgentServerHandle } from '@llui/agent/server'
import { createServer, type Server } from 'node:http'
import { AddressInfo } from 'node:net'
import { bundleHost } from './build.js'
import { createBridgeServer } from 'llui-agent/dist/bridge.js'  // tap into bridge internals
import { BindingMap } from 'llui-agent/dist/binding.js'
import { Client } from '@modelcontextprotocol/sdk/client/index.js'
import { InMemoryTransport } from '@modelcontextprotocol/sdk/inMemory.js'  // name may differ; check SDK

export type E2EContext = {
  browser: Browser
  page: Page
  server: Server
  httpPort: number
  agent: AgentServerHandle
  mcpClient: Client
  mintToken: () => Promise<{ token: string; wsUrl: string; lapUrl: string; tid: string }>
  bindClaude: (url: string, token: string) => Promise<void>
  close: () => Promise<void>
}

export async function setup(): Promise<E2EContext> {
  const bundle = await bundleHost()

  const agent = createLluiAgentServer({
    signingKey: 'x'.repeat(32),
    tokenStore: new InMemoryTokenStore(),
    identityResolver: async () => 'e2e-user',
    auditSink: { write: () => {} },
  })

  const server = createServer(async (req, res) => {
    // Serve bundle at /app.js and HTML at /
    if (req.url === '/') {
      res.writeHead(200, { 'content-type': 'text/html' })
      res.end('<!doctype html><html><body><div id="app"></div><script src="/app.js"></script></body></html>')
      return
    }
    if (req.url === '/app.js') {
      res.writeHead(200, { 'content-type': 'application/javascript' })
      res.end(bundle)
      return
    }
    // Delegate /agent/* to the agent router
    const webReq = new Request(`http://localhost${req.url}`, {
      method: req.method,
      headers: req.headers as Record<string, string>,
      body: ['GET', 'HEAD'].includes(req.method ?? 'GET') ? undefined : await readBody(req),
    })
    const webRes = await agent.router(webReq)
    if (!webRes) { res.writeHead(404); res.end(); return }
    webRes.headers.forEach((v, k) => res.setHeader(k, v))
    res.writeHead(webRes.status)
    res.end(await webRes.text())
  })
  server.on('upgrade', agent.wsUpgrade)
  await new Promise<void>((r) => server.listen(0, () => r()))
  const httpPort = (server.address() as AddressInfo).port

  // Bridge + MCP client in-process
  const bindings = new BindingMap()
  const bridge = createBridgeServer({ sessionId: 'e2e', bindings, version: '0.0.0' })
  const [clientT, serverT] = InMemoryTransport.createLinkedPair()
  await bridge.connect(serverT)
  const mcpClient = new Client({ name: 'e2e-client', version: '0.0.0' }, { capabilities: {} })
  await mcpClient.connect(clientT)

  const browser = await chromium.launch({ headless: true })
  const page = await browser.newPage()
  await page.goto(`http://localhost:${httpPort}/`)
  await page.waitForFunction(() => !!(window as any).__lluiE2eClient, null, { timeout: 5000 })

  const mintToken = async () => {
    const res = await fetch(`http://localhost:${httpPort}/agent/mint`, { method: 'POST' })
    const body = await res.json() as { token: string; wsUrl: string; lapUrl: string; tid: string }
    // Drive the browser's agent client to open the WS (via exposed global)
    await page.evaluate(async (b) => {
      const client = (window as any).__lluiE2eClient
      await client.effectHandler({ type: 'AgentOpenWS', token: b.token, wsUrl: b.wsUrl })
    }, { token: body.token, wsUrl: body.wsUrl })
    return body
  }

  const bindClaude = async (url: string, token: string) => {
    await mcpClient.callTool({
      name: 'llui_connect_session',
      arguments: { url, token },
    })
  }

  const close = async () => {
    await browser.close()
    await new Promise<void>((r) => server.close(() => r()))
    await mcpClient.close()
  }

  return { browser, page, server, httpPort, agent, mcpClient, mintToken, bindClaude, close }
}

async function readBody(req: import('node:http').IncomingMessage): Promise<Buffer> {
  const chunks: Buffer[] = []
  for await (const c of req) chunks.push(c as Buffer)
  return Buffer.concat(chunks)
}
```

Tests import `setup()` and call it in `beforeEach`. `afterEach` calls `close()`.

## Task 4: Core e2e tests

**Files (each is one test file with 1-3 cases):**
- `test/mint.e2e.test.ts` — mint token + WS pairs + describe returns app info
- `test/describe.e2e.test.ts` — describe_app returns the cached hello with docs
- `test/send-message.e2e.test.ts` — inc dispatches + state updates; signOut is humanOnly-blocked
- `test/confirm.e2e.test.ts` — delete proposes → browser shows pending → approve → dispatched
- `test/wait.e2e.test.ts` — wait_for_change times out without state changes; resolves when change occurs
- `test/dom.e2e.test.ts` — query_dom returns data-agent elements; describe_visible_content returns outline
- `test/revoke.e2e.test.ts` — revoke closes WS; subsequent LAP calls fail with revoked

Each test follows this pattern:

```ts
import { describe, it, expect, beforeEach, afterEach } from 'vitest'
import { setup, type E2EContext } from '../src/harness.js'

let ctx: E2EContext
beforeEach(async () => { ctx = await setup() })
afterEach(async () => { await ctx.close() })

describe('e2e: send_message', () => {
  it('dispatches inc and state updates', async () => {
    const mint = await ctx.mintToken()
    await ctx.bindClaude(mint.lapUrl, mint.token)

    const result = await ctx.mcpClient.callTool({
      name: 'send_message',
      arguments: { msg: { type: 'inc' } },
    })
    const content = JSON.parse((result.content[0] as { text: string }).text)
    expect(content.status).toBe('dispatched')

    const state = await ctx.page.evaluate(() =>
      (window as any).__lluiE2eHandle.getState(),
    )
    expect(state.count).toBe(1)
  })

  it('rejects humanOnly variants', async () => {
    const mint = await ctx.mintToken()
    await ctx.bindClaude(mint.lapUrl, mint.token)
    const result = await ctx.mcpClient.callTool({
      name: 'send_message',
      arguments: { msg: { type: 'signOut' } },
    })
    const content = JSON.parse((result.content[0] as { text: string }).text)
    expect(content.status).toBe('rejected')
    expect(content.reason).toBe('humanOnly')
  })
})
```

The confirm test is the most complex: after proposing, read the browser's `agentConfirm.pending` via `page.evaluate`; call `handle.send` to dispatch `Approve(id)`; poll for the MCP tool result to resolve.
