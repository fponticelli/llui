# 14. Compile-Time Rules

> **Auto-generated by `scripts/generate-rule-docs.ts`.** Re-run after rule changes:
> `pnpm tsx scripts/generate-rule-docs.ts`
>
> Single source of truth: `packages/compiler/src/lint-modules.ts`. Adding or removing a rule there propagates to both the transform pipeline and this doc.

Every rule listed here fires as a **compile-time error** from `@llui/compiler`, surfaced by `@llui/vite-plugin`'s build pipeline as `this.error(…)`. There is no warning tier; an LLM-first authoring loop treats anything else as ignorable noise.

## Categories

### `reactivity` (17)

Catch patterns that silently break reactivity — stale captures, side effects in accessors, missing reconciliation, impure reducers.

| Rule                                  | Description                                                                                                      |
| ------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| `llui/accessor-side-effect`           | Side effect (console, fetch, alert) inside a reactive accessor — accessors re-run on every state change.         |
| `llui/async-update`                   | update() must be synchronous and pure. Move async operations to effects.                                         |
| `llui/direct-state-in-view`           | Direct `state.X` read in an event handler — captures a stale snapshot. Use an accessor.                          |
| `llui/each-closure-violation`         | Identifier captured from parent scope inside each() render callback at a reactive position.                      |
| `llui/exhaustive-update`              | update() switch does not handle every variant of the file-local Msg union.                                       |
| `llui/imperative-dom-in-view`         | Imperative DOM access (document.querySelector, getElementById, etc.) in view() — use LLui primitives or onMount. |
| `llui/map-on-state-array`             | Array .map() on a state-derived value in view(). Use each() for reactive lists.                                  |
| `llui/nested-send-in-update`          | send() called inside update() — return effects instead.                                                          |
| `llui/no-eager-item-accessor`         | Eager call of an ItemAccessor (`text(item.x())`) captures a static value — drop the `()` to pass the accessor.   |
| `llui/no-let-reactive-accessor`       | `let` / `var` accessor referenced at a reactive position — compiler falls back to FULL_MASK. Use `const`.        |
| `llui/no-list-render-in-sample`       | `sample()` wrapping `.map()` over state-derived items — cells go stale.                                          |
| `llui/no-sample-in-accessor`          | `sample()` inside an accessor — invisible to mask analysis, breaks reconciliation.                               |
| `llui/no-sample-in-reactive-position` | `text(sample(…))` / `unsafeHtml(sample(…))` reads once at view-construction — DOM never updates.                 |
| `llui/pure-update-function`           | Side effect inside update() — return an Effect instead, keep update() pure.                                      |
| `llui/state-mutation`                 | Direct mutation of state in update() — state is immutable, return `{ ...state, field: value }`.                  |
| `llui/static-items`                   | `each({ items })` factory reads no state — list never reconciles.                                                |
| `llui/static-on`                      | `scope`/`branch` `on` accessor reads no state — key never changes.                                               |

## Rule severity

All compile-time rules emit at `severity: 'error'`. The LLui project's stance: warnings get reported and never fixed, so anything ship-worthy enough to ship at all ships as an error.

Adapters (e.g. `@llui/vite-plugin`) may downgrade to warnings on a per-host basis if needed (see `Diagnostic.severity` for the intent the rule itself emits). The vite-plugin currently calls `this.error()` for `error`, `this.warn()` for `warning`.

## Cross-file coverage

The agent rules (`agent-emits-drift`, `agent-msg-resolvable`) read imported `Msg` unions when the host adapter supplies the cross-file source via `typeSources.msg`. The vite-plugin does this automatically. Tests that invoke `transformLlui` directly without `typeSources` fall back to file-local-only behaviour.
