# Module decomposition — mapping

This document maps every file in `packages/compiler/src/` and every section of `transform.ts` (5 545 lines) to its target compiler module. It's the contract for the eventual decomposition work: when files leave this monolith they go to the slot named here. See `docs/proposals/v2-compiler/v2c.md` §2.2 for the four-module design and §7.9 for the handover that motivated this paper-only pass.

Scope: paper-only. The actual move happens in v2c/2.3 (proof-of-concept extraction) plus a future push that decomposes `transform.ts`. The decomposition is gated on the `CompilerModule` interface (v2c/2.2) landing first — without the visitor-registry primitive, the monolith can't be split without producing two AST passes over the same file.

After the move:

- `packages/compiler/` (umbrella) owns the entry point + module registry + shared utilities listed above.
- `packages/compiler-core/` owns reactivity, mask injection, structural primitives, `__update`/`__handlers`/`__prefixes` emission, template clone, row factory.
- `packages/compiler-agent/` owns `__msgSchema`, `__msgAnnotations`, `__bindingDescriptors`, agent diagnostics, the schema-hash + cross-file Msg/Effect-alias chase.
- `packages/compiler-ssr/` owns `__renderToString` emission + `'use client'` directive handling.
- `packages/compiler-devtools/` owns dev-only instrumentation hooks (`_eachDiffLog`, `_disposerLog`, `_effectTimeline`, `_coverage`).

## Cross-module dependencies

The activation graph (`v2c.md` §2.4 — modules declare deps on each other):

```
shared (always loaded, not user-facing)
   ↑
core (always on; declares no deps)
   ↑   ↑   ↑
   |   |   |
agent  ssr  devtools  (each declares core as a dep)
```

- **agent** depends on core: agent's substitution layer reads core's path collection; agent's diagnostic IDs reference core's emission slots.
- **ssr** depends on core: `__renderToString` needs core's view-tree shape.
- **devtools** depends on core: instrumentation hooks are inserted at core-emitted node positions.
- All three opt-in modules are mutually independent of each other.

The activation graph is enforced at compiler init: a config that names `agent()` without `core()` is a hard error (`llui/module-missing-dep`), naming both `core()` and the dependency.

## Validation pass

Before any decomposition code is written, walk one fixture component through this mapping by hand:

**Fixture:** the minimal `examples/counter/src/main.ts` (60-line component with state, msg, update, and a view that uses `text()` + `button()`).

Hand-trace per the registry's dispatch order:

1. **core** visitors on `CallExpression` nodes — see `text`, `button`, register reactive accessors, build path set, inject `__prefixes`, rewrite element calls.
2. **agent** visitors on `TypeAliasDeclaration` + `component()` call — see the inline `Msg` union, extract schema, emit `__msgSchema` + `__msgAnnotations` + `__schemaHash` + `__effectSchema`.
3. **devtools** visitors on the same `component()` call — emit `_eachDiffLog` / `_disposerLog` hook insertions (dev-only).
4. **umbrella** merges per-module emissions into the single transformed `component({...})` call.

The expected emission matches today's `transformLlui` output byte-for-byte (modulo property order, which is deterministic by §6.4). When the hand-trace produces a divergence, the mapping is wrong — fix before code lands.

## Open issues this mapping surfaced

1. **`cross-file-resolver.ts` is misnamed.** It does Msg-alias chasing only, not general cross-file resolution. Either rename to `msg-alias-resolver.ts` (agent module) or genuinely refactor into a core `cross-file-symbols.ts` + agent-specific consumer. The v2c §2 split is the natural moment to do this.
2. **`schema-hash.ts` straddles concerns.** Its inputs (msg schema, msg annotations) come from agent, but its consumer (HMR signaling at runtime) is umbrella-level. Decision: keep the function in agent, but the _call_ lives in the umbrella orchestrator after agent emission completes.
3. **`__lluiCompilerEmitted` vs. `__compilerVersion`.** Both ship from the same injector today. Both belong in umbrella (always-on integrity + version stamp). Split the injector into two functions during the move.
4. **`__bindingDescriptors` is fragile under module decomposition.** The connect-pattern injector mutates the AST before the main per-file walker runs (today, in `transform.ts` near the top). Under the module model, this is agent's pre-pass — but pre-passes break the §2.1 "walker runs once per file" invariant unless we explicitly allow modules to declare _transform passes_ in addition to _visitors_. Decide during v2c/2.2: either (a) the registry supports a `preTransform` hook, or (b) the connect-pattern pre-pass moves into agent's visitor and the AST mutation happens inline (more disruptive but keeps the invariant).
